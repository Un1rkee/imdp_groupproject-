Java API

API (Application Programming Interface, Application Programming Interface) is some predefined functions, the purpose is to provide applications and developers based on a software or hardware to access a set of routines, without having to access the source code, or understand Details of the working mechanism.

When running a Java program, the Java API class file used by the class file of the virtual machine loader. All the loaded class files (including those extracted from the application and from the Java API) and all the loaded dynamic libraries (including native methods) form the entire program running on the Java virtual machine. Before a platform can support Java programs, API functions must be explicitly implemented on this particular platform. To access local resources on the host, the Java API calls local methods. Since Java API class files call native methods, Java programs do not need to call them anymore. In this way, the Java API class file provides a platform-independent, standard interface Java program for the underlying host. For Java programs, regardless of the internal platform, the Java API will have the same performance and predictable behavior. It is precisely because the Java virtual machine and Java API are clearly implemented on each specific host platform that the Java program itself can become a platform-independent program.
